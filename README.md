# 🧬 BLAST 數據分析與程式架構重構專案

本專案旨在將基礎的數據處理腳本演進為具備「模組化 (Modularity)」與「穩健性（Robustness）」的專業程式。

## 🎯 專案階段目標
1. 目前的腳本 `rawCodes1142_2.py` 僅能處理基礎、小規模且格式完美的數據。本練習請進行「拆裝與重構」，以應對未來**更複雜、更大量且具有異常值**的類似數據。
2. 在課堂檢查時，需因應教師隨機要求以解釋或更改程式小參數。

### 1. 模組化開發要求 (不可一鍵式產出)
嚴禁將所有邏輯塞在單一檔案。必須將程式拆解為：
* **數據解析模組**：處理檔案讀取、類型轉換與異常過濾。
* **運算分析模組**：執行邏輯判斷與數據篩選。
* **視覺化模組**：負責高品質圖表的產出。
* **主入口 (main.py)**：調度上述模組。
並在 main.py 及各模組的開頭，用寫下 「這段程式碼的邏輯流程」。

> **⚠️ 警告**：禁止直接提交 AI 生成的「單一檔案全功能程式」。學生需展示如何將 AI 的片段建議整合進自己的模組架構中。

### 2. 應變能力測試 (之後將逐步更新)
本專案預計會有「壓力測試」。課堂將於後續釋出**測試檔案**。你們的程式架構必須能透過「修改參數」或「增加篩選邏輯」快速應變，而非重寫整套程式。
